# ETHICAL CYBERHACKERS

# CYBERSECURITY INTERN CURRICULUM REBUILD

# MASTER REPLIT CURRICULUM PROMPT

## OBJECTIVE

Refactor the Cybersecurity Intern curriculum into a realistic cybersecurity career simulation that teaches:

* Investigation
* Analytical thinking
* Risk assessment
* Decision making
* Business consequences
* Professional cybersecurity workflows

The curriculum must not become:

* Linux training
* Command memorization
* Quiz-based learning
* Flashcard learning

Students should feel like cybersecurity professionals investigating real incidents.

---

# CORE CURRICULUM PRINCIPLE

The student should learn:

```text
What happened?
Why does it matter?
How serious is it?
What should happen next?
```

Not:

```text
What command should I memorize?
```

Commands are tools.

Investigation is the goal.

---

# CURRICULUM PROGRESSION MODEL

Difficulty should increase through:

* More evidence
* More ambiguity
* More conflicting indicators
* More business consequences
* More decision complexity

Difficulty should NOT increase through:

* More jargon
* More unexplained logs
* More complex commands
* More technical terminology

---

# INVESTIGATION DESIGN RULE

Every command must contribute to answering:

1. What happened?
2. Who did it?
3. How serious is it?
4. What should we do next?

If a command does not contribute to one of these questions, remove it.

---

# COMMAND LEARNING LOOP

Every command should follow:

Command
↓
Raw Evidence
↓
Observation
↓
Interpretation Question
↓
Risk Assessment
↓
Recommendation
↓
Feedback
↓
Analyst Notebook Update

Students should never stare at logs without guidance.

---

# EVIDENCE DISPLAY SYSTEM

Every evidence item must support:

## Beginner View

Plain-language explanation.

## Analyst View

Semi-technical explanation.

## Technical View

Professional cybersecurity terminology.

Cybersecurity Intern defaults to:

Beginner View

Analyst View and Technical View remain expandable.

---

# TERMINOLOGY RULE

Never introduce terminology before behavior.

Bad:

"Beaconing"

Good:

"This computer contacts the same internet server every minute."

Student identifies behavior.

Then unlock:

Analyst Term:
Beaconing

Definition:
Repeated communication between systems at regular intervals.

---

# ANALYST NOTEBOOK

Create a persistent notebook.

Every command automatically adds findings.

Example:

Evidence Collected

✓ Unknown device discovered

✓ Contractor account accessed restricted files

✓ Multiple failed logins followed by success

✓ Large customer data transfer

Potential Risks

□ Unauthorized Device

□ Compromised Account

□ Data Exfiltration

Recommended Actions

□ Escalate Incident

□ Recommend Account Lock

□ Recommend Legal Review

Notebook updates throughout mission.

# END GOAL

Students should complete the Cybersecurity Intern curriculum feeling:

"I investigated real cybersecurity incidents."

Not:

"I memorized cybersecurity terminology."

The curriculum should progressively expose realistic evidence, realistic logs, realistic workflows, and realistic consequences while continuously supporting student understanding through guided investigation and reasoning.
