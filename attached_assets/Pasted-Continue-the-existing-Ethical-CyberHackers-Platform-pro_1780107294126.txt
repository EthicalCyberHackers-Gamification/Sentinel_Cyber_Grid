## FIX 1 — SLOW TERMINAL TYPING

Reduce the terminal typing speed so beginner students can follow the output.

Requirements:

* Slow down command/output animation.
* Use readable pacing.
* Add a configurable constant such as:

TERMINAL_TYPE_SPEED = 25 to 40 milliseconds per character

or line-based pacing:

TERMINAL_LINE_DELAY = 400 to 700 milliseconds

* Make sure terminal output is not instant unless the user skips animation.
* Add optional “Skip Animation” behavior if easy:

  * clicking terminal while typing completes the current output immediately.

Goal:
Students should be able to read commands and outputs as they appear.

## FIX 2 — EMPLOYEE NOTES CLASSIFICATION

Review the evidence classification logic for employee_notes.txt.

Current problem:
Employee Notes shows:
“Your earlier call didn’t fit. Re-judge this finding.”

This may be confusing because Employee Notes is likely supporting evidence, not a threat.

Expected Mission 1 classification logic:

employee_notes.txt:
Correct classification:
Helpful Supporting Evidence

meeting_schedule.txt:
Correct classification:
Normal Activity

finance_update.txt:
Correct classification:
Normal Activity

security_policy.txt:
Correct classification:
Helpful Supporting Evidence

suspicious_file.txt:
Correct classification:
Critical Threat Evidence

Fix the classification validation so Employee Notes is accepted as Helpful Supporting Evidence.

If the student classified Employee Notes correctly, show:
“Good. This note supports proper reporting behavior.”

If the student classified Employee Notes incorrectly, show:
“Re-check this note. It supports reporting suspicious files, but it is not the main threat.”

Do not make Employee Notes appear like an error if it was classified correctly.
