Yes. All of those changes are possible at the design level, and most are also straightforward at the UI implementation level. The bigger issue is not just the notebook's size or color treatment - it's that players are using it as a high-friction task manager instead of a satisfying investigation tool.

Core diagnosis

The Analyst Notebook sounds like it's carrying too many jobs at once:

• storing general world/context info
• surfacing active questions
• tracking urgent tasks
• logging events
• helping the player think

When one panel does all of that in the same visual language, it becomes work instead of play. Players stop scanning meaningfully and start cleaning up clutter.

Direct answers to your questions
1) Force non-urgent sections to collapse as the game proceeds

Yes - and this is a good idea, if done carefully.

Best version:
• Auto-collapse resolved or low-priority sections
• Keep active investigations pinned open
• Let players manually re-open anything
• Use a small label like Archived, Background, or Passive Lead

A strong rule set could be:

| Section type | Default state |
|---|---|
| Critical / time-sensitive | Expanded |
| Active inquiry | Expanded |
| Optional side lead | Collapsed |
| Resolved item | Collapsed |
| Lore / background info | Collapsed |

Important caution: do not make players feel the game is hiding information from them. Auto-collapse should feel like smart organization, not loss of control.

A good compromise is:
• auto-collapse after a stage transition
• brief highlight when something moves
• one-click Expand All
• one-click Show New / Active Only

2) Make inquiries and events/questions clearly different from general information

Absolutely - this is probably the highest-impact improvement.

Right now, the pain sounds like insufficient information hierarchy. Players need to instantly distinguish:

• facts
• suspicions
• questions
• new events
• urgent actions
• resolved conclusions

You can separate them through text treatment, color, iconography, framing, and motion.

A useful pattern:

| Content type | Visual treatment |
|---|---|
| General info | Neutral background, standard body text |
| Inquiry / open question | Accent color, question icon, bordered card |
| New event | Highlight band, timestamp, subtle pulse/new marker |
| Urgent issue | Strong contrast, warning color, pinned to top |
| Resolved conclusion | Muted tone, checkmark, collapsed by default |

Recommended visual language
• General information  
  Soft neutral panel, lower contrast, smaller header weight

• Inquiry / question  
  Distinct card style with a clear ? symbol, brighter accent, maybe a notebook-tab shape

• Event / incident update  
  Timestamped chip, different background tone, maybe left-edge color bar

• Urgent / critical  
  Reserved color such as amber or red, but used sparingly

• Resolved  
  Greyed slightly, checkmark, moved to archive/history group

The key is not just color. Use multiple cues at once:
• color
• icon
• header label
• card shape
• spacing
• ordering

That makes the UI readable for more players, including those less sensitive to color differences.

3) Increase the height of the Analyst Notebook

Yes, and you probably should test this.

If this panel is the player's main reasoning surface, a cramped vertical space will make everything worse:
• more scrolling
• less context visible at once
• weaker comparison between clues
• more cognitive fatigue

Good options:
• increase default height
• make it resizable
• allow pop-out / focus mode
• support full-screen notebook mode
• preserve the last used size

Best practical solution:
• standard docked mode for normal play
• expanded investigation mode when the player wants to concentrate

That makes the notebook feel like a tool the player chooses to use, not a box they're trapped inside.

Changes that would make it easier to work with
1) Split the notebook into clearer tabs or lanes

Instead of one mixed feed, divide it into:

• Active Leads
• Questions
• Evidence / Facts
• Events / Updates
• Resolved
• Background / Lore

Even two-column organization would help:
• left: active questions and tasks
• right: supporting notes and evidence

2) Add a priority and status system

Each note should communicate status instantly:

• New
• Active
• Blocked
• Optional
• Urgent
• Resolved

This reduces reading load because players don't need to interpret every line from scratch.

3) Show "what changed" clearly

A notebook often feels like a chore because players keep rereading old entries to find the one meaningful update.

Fix that with:
• NEW marker
• "Updated with 2 new clues"
• highlighted changed lines for a short time
• jump-to-latest button
• sort by recent update

4) Reduce text density

If entries are long, players will skim badly or ignore them.

Use:
• short summary line first
• expandable detail underneath
• bullets for clues
• highlighted keywords
• one idea per card

Good pattern:
• Headline: "Phone records contradict witness timeline"
• Why it matters: short sentence
• Details: collapsed unless opened

5) Add filtering

Let players toggle:
• Active only
• Unresolved only
• Events only
• Questions only
• Most recent
• Story-critical only

This is one of the easiest usability wins.

6) Pin and compare

Players doing analysis often want to keep 2-3 things visible together.

Useful features:
• pin note
• side-by-side compare
• reference while browsing
• cross-link related entries

That makes the notebook feel investigative instead of administrative.

How to make it more immersive and less of a chore

This is the most important part.

The notebook should not feel like a backlog. It should feel like the player's investigative mind made visible.

Strong immersion ideas
1) Turn notes into deductions, not storage

Instead of only listing information, the notebook should actively frame meaning:

• "This witness may be lying."
• "Timeline gap: 8:00-8:30 PM."
• "The key may connect the warehouse and the accountant."

That transforms the notebook from archive to analysis engine.

2) Use progressive reveal

Don't dump all note detail immediately.

Let notebook items evolve:
• initial clue
• then contradiction appears
• then hypothesis updates
• then conclusion locks in

This gives entries a sense of narrative motion.

3) Make the notebook react to player behavior

For example:
• if the player ignores a critical lead, it becomes visually more insistent
• if several clues support the same theory, they cluster together
• if a question is answered, the notebook visibly closes that thread

This makes the notebook feel alive.

4) Add relationship mapping

A simple visual link system can make analysis much more satisfying:
• person to event
• event to evidence
• evidence to question
• contradiction markers

Even lightweight graph behavior can create a much stronger detective fantasy.

5) Reward insight, not housekeeping

If the notebook mainly asks players to read and manage text, it feels like admin.

If it helps them notice patterns and make deductions, it becomes compelling.

Possible systems:
• player assembles clue links
• player confirms a hypothesis
• player chooses which lead to pursue
• notebook generates a deduction when enough evidence accumulates

6) Give the notebook a tone and identity

If appropriate for the game, the notebook itself can have character:
• field dossier aesthetic
• analyst terminal aesthetic
• marked-up case file aesthetic
• surveillance board aesthetic

Even subtle presentation changes help:
• stamped tags
• typed + annotated layers
• redacted or flagged entries
• audible paper/terminal feedback

That creates emotional texture.

A strong redesign direction

If I were simplifying this system, I'd move toward this structure:

Proposed notebook structure
Top bar
• Search
• Filter
• Active only toggle
• Expand/Collapse all
• Investigation mode button

Main sections
Critical Now - urgent, always visible
Open Questions - distinct visual treatment
Active Leads - current actionable threads
Evidence - facts and discovered clues
Recent Events - latest updates
Resolved Threads - collapsed archive
Background - collapsed by default

Entry anatomy
Each notebook item should have:
• title
• type label
• urgency/status
• short summary
• last updated marker
• expandable details
• related links

Example of better differentiation

Instead of this:

• Interview notes
• Warehouse info
• Missing key
• Timeline issue
• New phone call
• Background on suspect

Use this:

• OPEN QUESTION - Where was the missing key between 7 and 9 PM?
• NEW EVENT - Unidentified call logged from the warehouse line
• ACTIVE LEAD - Re-check warehouse access records
• FACT - Suspect badge used at 8:12 PM
• RESOLVED - Interview alibi confirmed false
• BACKGROUND - Company ownership history

Now the player can scan the panel in seconds.

Best-priority changes if you want the fastest improvement

If you only change five things, I would do these first:

Separate questions/events from general info visually
Collapse background and resolved items by default
Increase notebook height and add resizable/focus mode
Add status tags: New, Active, Urgent, Resolved
Highlight only changed/new content so players stop rereading everything

Those five alone would likely make a noticeable difference.

Design principle to keep in mind

The notebook should answer three player questions instantly:

• What matters now?
• What changed recently?
• What can I do with this information?

If the interface does not answer those in under a few seconds, it will keep feeling like a chore.

Bottom line

Yes:
• force-collapse of non-urgent sections: good idea
• clear visual distinction for inquiries/events/questions: essential
• increase notebook height: likely very worthwhile
• make it easier and more immersive: absolutely, by shifting from information dump to deduction-focused design

The real goal is to turn the Analyst Notebook from a passive text container into an active investigation companion.
