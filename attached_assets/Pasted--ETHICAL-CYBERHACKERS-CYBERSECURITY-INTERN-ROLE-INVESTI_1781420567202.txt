# REPLIT BUILD PROMPT

IMPORTANT

This is NOT a curriculum rewrite.

Do NOT rename missions.

Do NOT redesign the overall UI.

Do NOT remove:

* Investigation Terminal
* Analyst Notebook
* Resource Bar
* Investigation Confidence
* Sarah Reyes
* Existing mission flow

The goal is to transform passive investigations into active detective gameplay while preserving all existing learning objectives.

The biggest problem today:

The player discovers evidence but is rarely required to think about it.

Current flow:

Command
→ Read Output
→ Notebook Update
→ Next Command

Target flow:

Command
→ Discovery
→ Visual Reveal
→ Analyst Judgment
→ Feedback
→ Notebook Update
→ Continue Investigation

The player should repeatedly move through:

Discover
→ Interpret
→ Justify
→ Learn

This should feel similar to successful detective and investigation games such as:

* Return of the Obra Dinn
* Case of the Golden Idol
* Papers Please
* Her Story

The student should feel like they are solving a case, not reading a lesson.

---

## CORE DESIGN PRINCIPLE

The Investigation Terminal discovers facts.

The Visual Investigation Board reveals relationships.

The Analyst Notebook captures reasoning.

Sarah Reyes teaches judgment.

Each system has a unique role.

Avoid duplicate functionality.

---

## VISUAL INVESTIGATION BOARD

Repurpose the current in-mission Network Map.

For the Intern role, it becomes:

Visual Investigation Board

The board should not function as a traditional network diagram.

The board should function as a visual case board.

Every important discovery becomes a visual node.

Examples:

People
Devices
Files
Accounts
Systems
Alerts
Databases
Transfers

Nodes should visually connect as relationships are discovered.

The board should reveal the investigation gradually.

Students should feel they are uncovering a mystery.

---

## DISCOVERY EVENT SYSTEM

Do NOT open the board after every command.

Only trigger board events when:

1. New Entity Discovered
2. New Relationship Discovered
3. New Risk Discovered
4. Major Investigation Milestone Reached

Examples:

New Contractor Found

Unknown Device Found

Sensitive File Found

Compromised Account Found

Large Transfer Found

Each event should trigger a Visual Investigation Moment.

---

## VISUAL INVESTIGATION MOMENT

When a major discovery occurs:

The board slides open from the side.

Do NOT use intrusive modal popups.

The terminal remains visible.

The board highlights the new node.

Example:

Student runs:

cat release_notes.txt

Board reveals:

[ External Contractor ]
│
▼
[ Release Package ]

The relationship animates.

Student attention is drawn to the connection.

---

## ANALYST JUDGMENT CARDS

After a visual reveal:

Present a short judgment challenge.

Example:

ANALYST ASSESSMENT

What stands out?

○ Contractor involvement
○ File timestamp
○ File name
○ Nothing unusual

Student must answer.

Immediate feedback follows.

This is the core engagement mechanic.

The student should be making judgments throughout the mission.

Not only at the end.

---

## SARAH REYES MENTOR FEEDBACK

Avoid generic:

"Correct"

Use contextual mentor feedback.

Example:

Sarah Reyes:

"Good observation.

External contractors normally do not prepare release packages without internal review.

Let's find out who approved this."

Feedback should feel conversational and educational.

Sarah should act as a mentor.

Not a tutorial system.
