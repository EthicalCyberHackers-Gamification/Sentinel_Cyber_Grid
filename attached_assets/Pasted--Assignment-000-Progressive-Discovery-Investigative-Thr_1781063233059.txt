# Assignment 000 — Progressive Discovery & Investigative Thrill Refinement

## Dynamic SOC Investigation Experience

Refine Assignment 000 only.

Do not modify Assignment 001 or later assignments yet.

This refinement is based on the latest implementation screenshots and current gameplay behavior.

---

## CURRENT PROBLEM

The current network visualization reveals:

* too many devices
* too many traffic relationships
* the unknown infrastructure
* the suspicious path

too early.

This removes:

* curiosity
* investigative thrill
* gradual discovery
* analytical tension

Right now the player immediately sees:

* the suspicious infrastructure zone
* the unknown host
* many device relationships
* large portions of the final network picture

before earning that information through investigation.

The investigation should feel like:
discovering the network step-by-step.

NOT:
being shown the answer visually at mission start.

---

## CORE EXPERIENCE GOAL

Assignment 000 should teach:

1. Networks contain many communicating systems
2. Analysts identify what devices are communicating
3. Analysts determine whether communication is:

   * normal
   * approved
   * suspicious
   * off-baseline
4. Analysts slowly build confidence
5. Analysts escalate only after enough evidence exists

The player should feel:
“I uncovered the suspicious infrastructure myself.”

NOT:
“The game already showed me the suspicious infrastructure.”

---

## PRIMARY DESIGN PRINCIPLE

The map must follow the investigation.

The network picture should evolve only as evidence is discovered.

The visual layer should reward investigation.
