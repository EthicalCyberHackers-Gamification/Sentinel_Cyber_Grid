Apply this only to Assignment 000.

Do not modify Assignment 001 or later assignments yet.

Goal:
Make Assignment 000 more dynamic and visually useful by introducing simple network zones and live traffic visualization.

This should help beginners understand:

* what systems are communicating
* whether communication is internal or external
* what normal traffic looks like
* what suspicious traffic looks like
* how SOC analysts reason through network activity

This must be functional visualization, not decorative cyber graphics.

---

## CORE DESIGN PRINCIPLE

The visuals should answer:

What is communicating with what?
Should this communication be happening?
What changed as the investigation progressed?

Do not add meaningless animation.

Every visual element should support learning.

## NETWORK ZONES TO ADD

Create a simple visual network map with clear zones:

1. Internal User Zone
   Examples:

* WS-4471
* employee workstation

2. Internal Services Zone
   Examples:

* internal DNS
* file server
* identity service

3. External Internet Zone
   Examples:

* CDN
* vendor service
* unknown external host

4. Suspicious Infrastructure Zone
   Examples:

* unknown IP
* suspicious host
* possible scanning source

Keep the zones minimal and readable.

---

## PROGRESSIVE ZONE REVEAL

Do not show everything immediately.

Reveal the network progressively as the investigation advances.

Suggested flow:

Stage 1 — Triage:
Show only WS-4471 and limited traffic.

Stage 2 — SOC Tool Analysis:
Reveal external hosts and internal services.

Stage 3 — Correlation:
Separate known-good traffic from suspicious traffic.

Stage 4 — Escalation:
Highlight the suspicious source and show why it is worth escalation.

Stage 5 — Response / Consequence:
Show watchlist / monitoring status.

## TRAFFIC VISUALIZATION

Add subtle animated traffic pulses.

Examples:

* WS-4471 ↔ internal DNS
* WS-4471 ↔ CDN
* unknown external source → WS-4471

Rules:

* normal traffic should feel stable and calm
* suspicious traffic should feel irregular or isolated
* animation should be subtle
* avoid visual noise

The purpose is to teach communication relationships.

---

## BENIGN VS SUSPICIOUS TRAFFIC

Include at least three traffic types:

1. Normal internal traffic
   Example:
   WS-4471 → internal DNS

2. Normal external traffic
   Example:
   WS-4471 → known CDN

3. Suspicious or ambiguous traffic
   Example:
   203.0.113.77 → WS-4471 across multiple unrelated ports

The player should learn:
external does not automatically mean malicious.

The key question is:
Does the behavior match expected activity?

---

## BASELINE VISUALIZATION

Use the map to make baseline thinking visible.

Known-good systems:

* appear as trusted
* remain stable
* are labeled as expected after baseline comparison

Suspicious systems:

* remain uncertain at first
* become more suspicious as evidence accumulates
* are only highlighted after correlation

Do not label the suspicious source as malicious too early.

Use uncertainty first.
